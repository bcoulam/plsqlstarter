Table APP_LOCK

Logical Locks (ALK): Stores logical locks for applications that have pessimistic locking requirements.
Columns
Name Type Optional Default Comments
LOCK_ID NUMBER     Lock ID: Surrogate key for this table.
APP_ID NUMBER     Application ID: Foreign key to APP. The application which owns" the lock."
LOCK_NM VARCHAR2(1020)     Lock Name: Fixed name upon which parties involved in lock coordination have agreed. This could be the name of the screen, process, module, or simply the name of a table. This, combined with the ID or RID, provides the unique key to any given lock.
LOCKED_OBJ_ID VARCHAR2(240) Y   Locked Object ID: Primary key to the row of data to lock. Required for fine-grained locks.
LOCKED_OBJ_RID UROWID(4000) Y   Locked Object Rowid: The extended (ROWID) or logical rowid (UROWID) of the record being locked.
LOCKER_IP VARCHAR2(40) Y   Locker IP Address: IP address in IPv4 or IPv6 format of the client from which the user which obtained the lock. Use this value to inform concurrent lock requesters who they have to contact if they want to modify the data behind the lock.
LOCKER_ID VARCHAR2(256)     Locker ID: Any valid identifier for a system, process, account, role or user requesting the lock. There needs to be a way to track who is holding a lock, so this value needs to be as specific and traceable as possible.
LOCKED_DTM DATE     Lock Date and Time: When the lock was granted.
