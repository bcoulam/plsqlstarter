Table APP_MSG

Standard Messages (AM): Stores static messages and dynamic message templates used within the systems sharing Core. Static messages are repeated in logging targets as-is. Message templates have placeholder symbols that allow contextual substitution at runtime (which allows a higher degree of message re-use within and across systems).
Columns
Name Type Optional Default Comments
MSG_ID NUMBER     Message ID: Surrogate key for this table.
APP_ID NUMBER     Application ID: Foreign key to APP. Even though this column indicates which application owns" the message the row can and should be re-used by other applications if applicable. This is why the natural key does not include app_id."
MSG_CD VARCHAR2(240)     Message Code: Short code or name that identifies a message within a given subsystem.
MSG VARCHAR2(1020)     Message: The full text of the standard message or message template. If the message is a template with placeholders, surround the substitutable placeholders with the substitution character defined in the C package (default is @).
MSG_DESCR VARCHAR2(4000) Y   Message Description: If the message text contains placeholders, e.g. @1@ not allowed to read from table @2@" this field provides a place to describe what those placeholders mean. In the example provided here the MSG_DESC could be 1 = app.app_cd, 2 = table name being read.
MSG_SOLN VARCHAR2(4000) Y   Message Solution: Optional steps, hints, or actions the user might take to report or solve the problem on their own.
