Table APP

Applications (A): List of the applications or systems that wish to use the Core framework. In many cases, there will only be one system using Core. If you have multiple applications on the same database and they are small enough, you can create them in the same schema. If they are too large and complex, create them in separate schemas, layered on top of the Core schema using grants and synonyms.
Columns
Name Type Optional Default Comments
APP_ID NUMBER     Application ID: Surrogate key for this table.
APP_CD VARCHAR2(20)     Application Code: Short code that indicates the system or schema owning the row, thus allowing a logical separation of records in the framework tables that belong to disparate systems.
APP_NM VARCHAR2(200)     Application Name: Short name that indicates the system or schema owning the row, thus allowing a logical separation of records in the framework tables that belong to disparate systems.
APP_DESCR VARCHAR2(4000)     Application Description: In-depth description of the system, its purpose, sponsors, information stewards, etc.
